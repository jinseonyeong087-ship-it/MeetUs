# 화면설계서 (AA)

## 1. 목적
AI 회의 요약 및 개인별 To-Do 자동 추출 결과를 사용자에게 직관적으로 제공한다.

---

## 2. 화면 구성

## 2.1 회의 업로드 화면
- 파일 업로드 영역 (drag & drop / 파일 선택)
- 회의 제목 입력
- 업로드 시작 버튼
- 업로드/처리 진행률 표시
- 업로드 정책 안내 (`m4a`, 30MB 이하)

### 주요 상태
- 업로드 대기
- 업로드 중
- 업로드 완료
- 업로드 실패 (재시도)

---

## 2.2 회의 목록(Archive) 화면
- 검색창 (제목/참석자)
- 필터 (기간, 상태)
- 정렬 (최신순/오래된순)
- 페이지네이션
- 목록 카드/테이블 항목
  - 회의 제목
  - 생성일시
  - 처리 상태
  - 요약 생성 여부

---

## 2.3 회의 상세 화면
- 회의 기본 정보
- 전체 요약 섹션
- 결정사항 섹션
- 원문 transcript 섹션
- 개인별 To-Do 카드 섹션
  - 담당자
  - 할 일 내용
  - 마감일(선택)

### 예외 처리 UI
- 처리 실패 안내 메시지
- 재처리 버튼

---

## 3. 공통 UI 규칙
- 상태 뱃지: `CREATED`, `UPLOADED`, `PROCESSING`, `COMPLETED`, `FAILED`
- 로딩 스켈레톤 사용
- API 실패 시 토스트 + 재시도 버튼 제공

---

## 4. 반응형 기준
- Desktop 우선 설계
- Tablet/모바일에서 목록은 카드형으로 자동 전환
