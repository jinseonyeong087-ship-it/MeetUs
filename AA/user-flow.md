# 사용자 흐름도 (AA)

## 1. 메인 사용자 시나리오
1. 사용자가 워크스페이스 내에서 회의를 생성
2. 시스템이 Presigned URL을 발급
3. 사용자가 `m4a` 파일을 S3에 업로드
4. 백엔드가 업로드 완료 후 AI 처리 파이프라인 시작
5. 사용자는 목록/상세 화면에서 처리 상태 확인
6. 처리 완료 후 요약/개인별 To-Do 확인
7. Archive에서 검색/필터/정렬로 과거 기록 조회

---

## 2. 상태 전이 기준 사용자 흐름
- `CREATED` → 회의 메타데이터 생성 완료
- `UPLOADED` → 업로드 완료
- `PROCESSING` → 전사/요약/To-Do 처리 중
- `COMPLETED` → 상세 결과 조회 가능
- `FAILED` → 오류 안내 + 재처리

---

## 3. 화면 이동 흐름
- 업로드 화면 → (업로드 성공) → 회의 상세 화면
- 상세 화면 ↔ 목록(Archive) 화면
- 목록 화면 → 특정 회의 클릭 → 상세 화면

---

## 4. 예외 시나리오
- 업로드 실패: 실패 사유 표시 + 재업로드 유도
- AI 처리 실패: 상태 `FAILED` 노출 + 재처리 버튼
- 데이터 없음: 빈 상태(Empty State) 안내
