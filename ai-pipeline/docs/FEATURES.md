# 🎯 시스템 기능 명세서 (Functional Specifications)

## 1. 서비스 개요
본 시스템(AI-Minutes)은 회의 음성 파일을 비동기 파이프라인으로 처리하여 텍스트로 변환(STT)하고, 대형 언어 모델(LLM)을 통해 회의 요약 및 담당자별 할 일(To-Do)을 자동 추출하는 클라우드 네이티브 아카이브 플랫폼이다.

## 2. 사용자 인터페이스 (UI) 요구사항
사용자가 프론트엔드(화면)에서 직접 상호작용하는 기능 명세이다.

### 2.1. 회의 등록 및 음성 업로드
* **빠른 등록 폼:** 사용자는 '회의 제목', '담당자(드롭다운)', '우선순위(보통/높음)', '메모'를 입력하여 회의록 껍데기를 빠르게 생성할 수 있다.
* **다이렉트 업로드:** 'Drag & Drop' 또는 '파일 선택' 버튼을 통해 음성 파일(**최대 30MB, .m4a 포맷 한정 지원**)을 백엔드 서버 부하 없이 AWS S3로 직접 업로드한다.

### 2.2. 대시보드 및 상태 모니터링
* **상태 분포 보드:** 현재 시스템에 등록된 전체 회의록의 진행 상태(`CREATED`, `UPLOADED`, `PROCESSING`, `COMPLETED`, `FAILED`)를 건수와 함께 시각화하여 제공한다.
* **회의 목록 뷰:** 좌측 사이드바에 생성일자 역순으로 회의 목록(주간 제품 회의, 고객 인터뷰 회고 등)을 노출하며, 각 항목의 우측 상단에 현재 상태 뱃지를 표시한다.

### 2.3. 회의록 상세 뷰 (Detail View)
처리가 완료된(`COMPLETED`) 회의록 클릭 시 4가지 탭으로 정보를 제공한다.
* **Overview:** 회의의 핵심 요약 내용과 담당자별 To-Do 카드를 중점적으로 보여준다.
* **Timeline:** 전체 음성 원본 텍스트(Full-text)를 발화 시간 순서대로 나열하여 제공한다.
* **Templates / Reports:** (※ MVP 단계에서는 구조만 잡아두고 추후 고도화 예정)

## 3. 시스템 및 AI 동작 (System Behaviors) 요구사항
사용자 눈에 보이지 않는 백그라운드 서버(Core API & AI Engine)의 동작 명세이다.

### 3.1. 비동기 이벤트 파이프라인
* S3 업로드가 완료되면 Core API는 DB에 기본 정보를 저장하고, AWS SQS로 작업 지시 메시지를 발행하여 타임아웃(Timeout)을 방지한다.
* AI 서버(Python)는 SQS 메시지를 지속적으로 폴링(Polling)하여 작업을 낚아챈다.

### 3.2. AI 텍스트 변환 (STT) 및 데이터 정형화 (LLM)
* **음성 인식:** AWS Transcribe를 호출하여 다자간 회의 음성을 한국어 텍스트로 변환한다.
* **정형화 추출:** Amazon Bedrock (Claude 3) 프롬프트 엔지니어링을 통해 **전체 요약(5~7줄) 및 결정사항**을 생성하고, 프론트엔드가 즉시 화면에 렌더링할 수 있도록 담당자별 할 일을 JSON 포맷(예: `{"assignee": "민수", "task": "스프린트 계획", "due_date": null}`)으로 가공하여 DB에 분리 적재한다. (담당자는 Workspace Member 테이블 기준으로 자동 매핑 가능하도록 구성)

## 4. 작업 상태값 규격 및 전이 조건 (Status Flow)
각 상태는 엄격한 조건 하에 순차적으로 변경되며, DB 업데이트 즉시 프론트엔드에 반영된다.

1. `CREATED`: 회의록 껍데기만 생성된 초기 상태.
2. `UPLOADED`: 사용자가 S3에 파일을 올리고 백엔드 호출을 마친 상태. (SQS 발행 대기)
3. `PROCESSING`: AI 서버가 SQS 메시지를 수신하여 STT/LLM 분석을 진행 중인 상태.
4. `COMPLETED`: AI 엔진이 전송한 결과 데이터를 Core API가 수신하여 DB 4대 테이블에 완벽히 저장(Commit)한 최종 상태.
5. `FAILED`: 재시도(3회) 후에도 실패하여 Dead Letter Queue로 넘어간 에러 상태.
